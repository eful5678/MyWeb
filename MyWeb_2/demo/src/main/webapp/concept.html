#2 Database

INSERT / SELECT / UPDATE / DELETE

  • INSERT

  • SELECT

    • 표시할 대상 열과 대상 열을 포함하는 테이블 지정
    • 기본 문법
      SELECT * | {[DISTINCT] column | expression [alias], ...}
FROM table;
      1. '*' : 모든 열 선택
      2. DISCTINCT : 중복되는 열 생략
      • 값이 같은 열을 생략한다.
      1. column | expression : 지정한 열 또는 표현식
      2. alias : 선택된 열에 다른 머리글 이름 지정
    • Example
      SELECT * // 대상 열
FROM table; // 대상 열이 포함된 테이블
      1. table로 부터 모든 열을 선택

  • ORDER BY - 정렬

    • ASC : 오름차순 => 기본값
    • DESC : 내림차순
    • SELECT문 가장 끝에 위치
    • Example
      SELECT last_name, job_id, department_id, hire_date
FROM employees
ORDER BY hire_date;
      1. 정렬 기준 : hire_date, ASC(기본값)
      2. 테이블 : employees
      3. 선택열 : last_name, job_id, department_id, hire_date
    • 여러 열을 기준으로 정렬 -Example
      SELECT last_name, department_id, salary
FROM employees
ORDER BY department_id, salary DESC;
      1. 정렬 기준 : 1)department_id, 2)salary -> DESC
      2. 테이블 : employees
      3. 선택열 : last_name, department_id, salary

  • WHERE - 조건 : 선택되는 행 제한

    • 기본 문법
      SELECT * | {[DISTINCT] column | expression [alias], ...}
FROM table;
[WHERE condition(s)];
      1. WHERE : 조건에 맞는 행만 오도록 제한
      2. condition : 열 이름, 표현식, 상수 및 비교 연산자로 구성
    • Example
      SELECT employee_id, last_name, job_id, department_id
FROM employees
WHERE department_id = 90;
      1. 조건 : department_id 열의 값이 90 인 경우
      2. 테이블 : employees로 부터
      3. 선택열 : employee_id, last_name, job_id, department_id의 값을 가져와라

비교 조건

  • BETWEEN ...AND...

    • 조건 범위에 따라 행 출력
    • 지정 범위 : 하한(이하) AND 상한(이상)
    • Example
      SELECT last_name, salary
FROM employees
WHERE salary BETWEEN 2500 AND 3500;
      1. 조건 : salary가 2500 과 3500 사이
      2. 테이블 : employees로부터
      3. 선택열 : last_name, salary

  • IN(set)

    • 조건에 따른 값(특정값)이 목록에 있는지 확인
    • 조건이 문자인 경우 -> ''를 붙여준다
    • Example
      SELECT employee_id, last_name, salary, manager_id
FROM employees
WHERE manager_id IN (100, 101, 201);
      1. 조건 : manager_id가 100 또는 101 또는 201
      2. 테이블 : employees로부터
      3. 선택열 : employee_id, last_name, salary, manager_id

  • LIKE

    • 유효한 검색 문자열 값인 대체 문자를 사용하여 검색
    • 대/소문자 구분
    • 'S%' : S로 시작하는
    • '%S' : S로 끝나는
    • Example
      SELECT first_name
FROM employees
WHERE first_name LIKE 'S%';
      1. 조건 : first_name이 'S'로 시작하는 모든 것
      2. 테이블 : employees로부터
      3. 선택열 : first_name

  • IS NULL

    • NULL 여부 테스트
    • Example
      SELECT last_name, manager_id
FROM employees
WHERE manager_id IS NULL;

JOIN

  • 여러 테이블의 데이터를 질의할 때 사용
  • WHERE절에서 JOIN 조건 작성
    • 동일한 열 이름이 여러 테이블에 있는 경우 열 이름에 테이블 이름을 접두어로 붙인다.
    • 기본 키 및 외래 키 열에 존재하는 공통 값에 따라 한 테이블의 행을 다른 테이블의 행에 조인
    • 기본 문법
      SELECT table1.column, table2.column
FROM table1, table2
WHERE table1.column1 = table2.column2;
      1. 조건: table1의 column1과 table2의 column2가 같을 때
      2. 테이블: table1, table2
      3. 선택열: table1.column, table2.column
  • 등가조인
    • 단순조인 / 내부조인
    • Example
      SELECT employees.employee_id, employees.last_name,
        employees.department_id, departments
        department_id, departments.location_id
FROM employees, departments
WHERE employees.department_id = departments.department_id;
      1. 조건: employees 테이블의 department_id열과 departments 테이블의 department_id 열이 같을 때,
      2. 테이블: employees, departments로 부터
      3. 선택열: 위와같음
  • 비등가조인
    • 동등 연산자가 아닌 연산자를 포함하는 조인조건
    • Example
      SELECT e.last_name, e.salary, j.grade_level
FROM employees e, job_grades j
WHERE e.salary
      BETWEEN j.lowest_sal AND j.highest_sal;
      1. 조건: employees 테이블의 salary가, job_grades의 lowest_sal 열과 highest_sal 사이에 있을 때,
      2. 테이블: employees, job_grades로 부터
      3. 선택열: last_name(employees), salary(employees), grade_level(job_grades)
      • BETWEEN 구문을 사용할 경우 낮은 값을 먼저 지정해야한다.

#5 Network

IP

  • 인터넷 프로토콜의 역할
    • 패킷이라는 통신 단위로 지정한 IP주소에 데이터 전달
    • IP 패킷 정보
      • 출발지IP, 목적지IP, ...
      • 클라이언트와 서버가 서로 패킷 정보를 보낸다
    • IP 프로토콜의 한계
      • 비연결성
        • 패킷을 받을 대상이 없거나 서비스 불능 상태여도 패킷 전송 -> 패킷 소실
      • 비신뢰성
        • 중간에 패킷이 사라지는 경우
        • 패킷이 순서대로 안오는 경우
            1. Hello 2. World로 전송 -> 1. World 2. Hellow로 전송 받음
      • 프로그램 구분
        • 같은 IP를 사용하는 서버에서 통신하는 애플리케이션이 둘 이상인 경우
  • TCP & UDP
    • IP 스택의 4계층
      • HTTP, FTP : 애플리케이션 계층
        • 웹 브라우저, 네트워크 게임
        • SOCKET 라이브러리
      • TCP, UDP : 전송 계층
      • IP : 인터넷 계층
      • 네트워크 인터페이스 계층
        • LAN 드라이버, LAN 장비
    • 데이터 전송 프로세스
      1. 프로그램이 Hello, world! 메시지 생성
      2. SOCKET 라이브러리를 통해 전송 계층으로 전달
      3. TCP 정보생성, 메시지 데이터 포함
      4. IP 패킷 생성, TCP 데이터 포함
      5. LAN 카드 -> 인터넷 -> 서버
